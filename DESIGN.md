# 설계 문서 — 기존 투두리스트 앱의 문제점과 해결 방안

> ⚠️ 출처에 대한 정직한 안내: 이 분석은 실시간 웹 검색이 차단된 환경에서 작성되어,
> 2026년 1월까지의 학습 지식(널리 알려진 사용자 불만, 리뷰, 커뮤니티 피드백)을 기반으로 합니다.
> 최신 버전에서 일부 문제는 개선됐을 수 있습니다.

## 1. 기존 앱들의 대표적인 문제점

### ① 플랫폼 호환성 문제
| 앱 | 문제 |
|---|---|
| Things 3 | 애플 전용 — 안드로이드/윈도우 사용 불가, 게다가 기기별로 따로 구매 |
| Microsoft To Do | 플랫폼별 기능·UI 불일치, 맥 앱 완성도 낮음 |
| Apple 미리알림 | 윈도우/안드로이드에서 사실상 사용 불가 |
| Google Tasks | 데스크톱 전용 앱이 없음 (브라우저 구석 패널 신세) |

**→ 해결: PWA(웹 표준 앱).** 안드로이드·맥·윈도우 모두 같은 코드, 같은 UI, 같은 기능. 브라우저만 있으면 동작하고 앱으로 설치도 가능.

### ② 데이터 소유권·오프라인 문제
- 대부분 앱이 서버 계정 필수 → 서비스 종료/계정 정지 시 데이터 인질
- Notion, Any.do 등은 오프라인에서 매우 취약
- 내 데이터를 표준 형식으로 꺼내기 어려움 (벤더 락인)

**→ 해결: 로컬 우선(local-first).** 모든 데이터가 내 기기에 저장, 계정·서버 불필요, 완전 오프라인 동작. JSON 백업/복원 + 표준 ICS 내보내기로 락인 제로.

### ③ 가격 정책 불만
- Todoist: 미리알림·기간 설정 등 핵심 기능이 유료
- TickTick: 캘린더 월간 뷰가 유료
- Things 3: 맥 + 아이폰 + 아이패드 각각 별도 구매 (총 8만원 이상)
- 구독 피로감 (월 구독이 기본이 된 시장)

**→ 해결: 전 기능 무료.** 서버 비용이 없는 구조라서 가능. 캘린더 뷰, 반복 일정, 우선순위, 태그 전부 기본 제공.

### ④ 캘린더 연동 문제
- Google Tasks ↔ Google Calendar조차 통합이 어색함
- 대부분 앱이 캘린더 연동을 유료 기능으로 묶거나 단방향만 지원
- 특정 플랫폼(구글만, 애플만)만 지원하는 경우 다수

**→ 해결: 2중 연동 전략.**
1. **ICS 표준 (기본, 무설정)** — 구글/애플/아웃룩/네이버 등 모든 캘린더가 읽는 RFC 5545 표준으로 내보내기·가져오기
2. **구글 캘린더 API (선택)** — 본인 OAuth 키 입력 시 양방향 실시간 동기화 (가져오기 + 올리기)

### ⑤ 기능 과잉/UX 문제
- Notion: 투두 입력에 너무 많은 클릭, 무겁고 느림
- Any.do: 광고·업셀 팝업
- 복잡한 GTD 구조 강요 (학습 비용)
- 빠른 입력이 안 됨 — 날짜 지정에 클릭 3~4번

**→ 해결: 자연어 빠른 입력.** `내일 오후 3시 회의 #업무 @회사 !높음 매주` 한 줄이면 끝.
한국어 날짜(내일/모레/다음주 월요일/6월 20일), 시간(오후 3시 반), 우선순위, 반복, 태그, 프로젝트를 자동 인식. 입력 중 실시간 미리보기 제공.

### ⑥ 동기화 신뢰성
- 다수 앱에서 기기 간 동기화 충돌·유실 보고
- 서버 장애 시 앱 전체 마비

**→ 해결(현 버전의 트레이드오프):** 서버를 없애 동기화 장애 자체를 제거. 기기 간 이동은 JSON 백업/ICS로 수동, 또는 구글 캘린더를 허브로 활용. (자동 멀티기기 동기화는 향후 과제 — 아래 §3)

### ⑦ 기타 해결한 디테일
- 실수 삭제 → **실행 취소(Undo) 토스트**
- 반복 일정 완료 시 **다음 회차 자동 생성**
- 지연된 할 일이 사라지지 않고 **"⚠️ 지연됨" 그룹**으로 표시
- 다크 모드, 키보드 단축키(n: 입력, /: 검색, 1~4: 뷰 전환)
- 하위 할 일(서브태스크) + 진행률 표시

## 2. 아키텍처

```
index.html          앱 셸 (사이드바 + 리스트/캘린더 + 모달)
css/style.css       라이트/다크 테마, 반응형(모바일 사이드바 서랍화)
js/store.js         데이터 계층 — localStorage, CRUD, 반복 일정 로직
js/nlp.js           한국어/영어 자연어 파서
js/ics.js           RFC 5545 ICS 생성/파싱 (줄접기·이스케이프 포함)
js/calendar.js      월/주 캘린더 렌더러
js/gcal.js          구글 캘린더 양방향 동기화 (GIS + gapi, 지연 로드)
js/app.js           UI 글루 코드
sw.js               오프라인 캐싱 서비스 워커
manifest.webmanifest PWA 설치 정보
serve.ps1           의존성 없는 로컬 서버 (윈도우 PowerShell 내장 기능만 사용)
```

프레임워크·빌드 도구·npm 의존성 **0개** — 10년 뒤에도 그대로 열리는 순수 웹 표준 코드.

## 3. 향후 과제 (현 버전의 한계)

1. ~~**자동 멀티기기 동기화**~~ → **해결됨 (v2)**: GitHub 비공개 Gist 기반 자동 동기화(js/sync.js). 항목별 최신 수정 우선 병합 + tombstone으로 삭제 전파. 변경 4초 후 자동 푸시, 앱 시작/포커스 시 자동 풀.
2. **알림(미리알림)** — PWA 푸시 알림은 호스팅(HTTPS) + 권한 허용 필요. 로컬 알림 API 추가 예정 과제.
3. **애플 캘린더 직접 연동** — 애플은 공개 REST API가 없음. ICS로 우회 중. CalDAV(앱 암호 방식) 추가 가능.
4. **아웃룩 직접 연동** — Microsoft Graph API 추가 가능 (검증 부담 거의 없음). 현재는 ICS로 커버.
