# ✓ 모두의 할 일 — 투두리스트 & 캘린더

안드로이드 · 맥북 · 윈도우 PC에서 모두 동작하는 **오프라인 우선, 전 기능 무료** 투두리스트 + 캘린더 앱.
계정도, 서버도, 설치 프로그램도 필요 없습니다.

## 🚀 바로 실행하기 (윈도우, 이 PC)

```powershell
cd C:\Users\msm16_\Documents\todo
powershell -ExecutionPolicy Bypass -File serve.ps1
```

브라우저에서 **http://localhost:8741** 접속. (맥에서는 `python3 -m http.server 8741` 등 아무 정적 서버나 사용)

> `index.html`을 더블클릭해서 열어도 동작합니다 (file:// 모드 — 오프라인 캐싱만 비활성화).

## ✨ 주요 기능

- **자연어 빠른 입력** — 한 줄로 끝: `내일 오후 3시 회의 #업무 @회사 !높음 매주`
  - 날짜: `오늘` `내일` `모레` `다음주 월요일` `6월 20일` `2026-07-01`
  - 시간: `오후 3시` `9시 반` `15:30`  · 우선순위: `!높음 !보통 !낮음`  · 반복: `매일 매주 매달 매년`
  - `#태그` `@프로젝트` 자동 인식 + 입력 중 실시간 미리보기
- **캘린더 뷰** — 월/주 전환, 날짜 클릭으로 바로 추가
- **반복 일정** — 완료하면 다음 회차 자동 생성
- **하위 할 일, 프로젝트, 태그, 검색, 다크 모드, 삭제 실행취소**
- **단축키** — `n` 새 할 일 · `/` 검색 · `1~4` 뷰 전환

## 🔁 여러 기기 자동 동기화

GitHub 계정의 **비공개 Gist**를 저장소로 사용해 안드로이드·맥·윈도우 모든 기기에서 데이터가 자동으로 합쳐집니다. 서버 비용 0원, 데이터는 본인 계정에만 저장됩니다.

1. https://github.com/settings/tokens/new?scopes=gist&description=modu-todo-sync 접속
2. **gist 권한만 체크된 상태** 그대로 → 만료 기간 선택(예: 1년) → "Generate token" → 토큰 복사
3. 각 기기에서 앱의 `설정 → 여러 기기 자동 동기화`에 같은 토큰 입력 → 연결

동작 방식: 변경 후 4초 뒤 자동 저장, 앱을 열거나 창에 돌아올 때 자동으로 받아옴.
충돌은 **항목별 최신 수정 우선**으로 병합되고, 한 기기에서 삭제하면 다른 기기에서도 삭제됩니다.

> 보안: 토큰은 gist 권한만 가지므로 저장소·코드에는 접근할 수 없습니다. 토큰은 각 기기의 브라우저에만 저장됩니다.

## 🔗 캘린더 연동

### 방법 1 — ICS 표준 (설정 불필요, 모든 캘린더 호환) ✅ 권장
`설정 → ICS로 내보내기` → 받은 `.ics` 파일을 구글 캘린더 / 애플 캘린더 / 아웃룩 / 네이버 캘린더에서 가져오기.
반대로 다른 캘린더에서 내보낸 `.ics`를 `ICS 가져오기`로 불러올 수 있습니다.

### 방법 2 — 구글 캘린더 실시간 양방향 동기화 (선택)
본인 명의의 무료 API 키가 필요합니다 (아래 "준비물" 참고). 설정에 키를 입력하면:
- 구글 캘린더의 일정 → 앱 캘린더에 표시 (가져오기)
- 날짜 있는 할 일 → 구글 캘린더에 자동 등록 (올리기)

## 💻 진짜 프로그램으로 설치하기

### 윈도우 — 설치 파일(.exe)
[GitHub Releases](https://github.com/bokuyong/modu-todo/releases)에서 `ModuTodo-Setup-x.x.x.exe` 다운로드 → 실행. 끝.
직접 빌드하려면: `npm install` 후 `npm run dist` → `dist\ModuTodo-Setup-*.exe` 생성.

### 맥북 — 앱(.dmg)
맥용 빌드는 맥에서만 만들 수 있습니다 (애플 정책). 맥북에서 한 번만:
```bash
# Node.js 설치 (https://nodejs.org) 후
git clone https://github.com/bokuyong/modu-todo.git && cd modu-todo
npm install && npm run dist:mac
```
→ `dist/ModuTodo-1.0.0.dmg` 생성 → 열어서 Applications에 드래그.
서명 없는 앱이라 첫 실행 시 "확인되지 않은 개발자" 경고가 나오면: 앱 우클릭 → 열기.

### 안드로이드 — 앱(APK)
PWABuilder(마이크로소프트의 무료 도구)로 APK를 만들 수 있습니다:
1. https://www.pwabuilder.com 접속 → `https://bokuyong.github.io/modu-todo/` 입력 → Start
2. **Android** 패키지 선택 → "Generate" → APK 다운로드
3. 폰에 파일 복사 → 실행 (설정에서 "출처를 알 수 없는 앱 설치" 허용 필요)
- 구글 플레이 스토어에 올리려면 개발자 등록비 $25(1회)가 필요하므로, 개인용은 APK 직접 설치가 무료입니다.
- 더 간단한 대안: Chrome에서 위 주소 접속 → 메뉴 → **"홈 화면에 추가"** — 아이콘·독립 실행·오프라인까지 동일하게 동작합니다.

### 설치 없이 (PWA)
| 기기 | 방법 |
|---|---|
| 윈도우 | Chrome/Edge에서 접속 → 주소창 우측 **설치 아이콘** 클릭 |
| 맥북 | Chrome 설치 아이콘 또는 Safari → 파일 → **"Dock에 추가"** |

> 참고: 데스크톱 프로그램(Electron)에서는 구글 캘린더 OAuth 연동이 동작하지 않습니다 (구글이 https 주소만 허용). 기기 동기화(GitHub)와 ICS는 정상 동작하며, 구글 캘린더 연동만 웹 버전에서 사용하세요.

## 🧰 사용자가 준비해야 할 것

### ① 지금 당장 (필수 아님 — 없어도 로컬에서 동작)
- 모던 브라우저(Chrome/Edge/Safari)만 있으면 됩니다. **추가 준비물 0개.**

### ② 모든 기기에서 쓰려면 — 무료 호스팅 (10분 소요)
PWA 설치와 안드로이드 사용을 위해 HTTPS 주소가 필요합니다. 가장 쉬운 방법:
1. https://github.com 계정 생성 (이미 있다면 생략)
2. 새 저장소(repository) 생성 → 이 폴더의 파일 전부 업로드
3. 저장소 Settings → Pages → Branch를 `main`으로 설정 → 저장
4. 몇 분 뒤 `https://<아이디>.github.io/<저장소이름>/` 주소 완성
- 대안: Netlify(drag&drop 배포), Cloudflare Pages, Vercel — 모두 무료

### ③ 구글 캘린더 실시간 동기화를 쓰려면 (15분 소요, 무료)
1. https://console.cloud.google.com 접속 → 새 프로젝트 생성
2. **API 및 서비스 → 라이브러리 → Google Calendar API → 사용 설정**
3. **API 및 서비스 → 사용자 인증 정보 → 사용자 인증 정보 만들기**
   - **API 키** 생성 → 복사해두기
   - **OAuth 클라이언트 ID** 생성 (유형: 웹 애플리케이션)
     - "승인된 자바스크립트 원본"에 앱 주소 추가: `http://localhost:8741` 그리고 ②에서 만든 https 주소
4. **OAuth 동의 화면** 구성: User Type = 외부, 게시 상태 = **테스트**, 테스트 사용자에 본인 Gmail 추가
5. 앱의 `설정 → 구글 캘린더 동기화`에 클라이언트 ID와 API 키 입력 → 연결
- 참고: 테스트 모드는 본인 전용으로 충분하며 앱 심사가 필요 없습니다. 토큰이 주기적으로 만료되면 "연결" 버튼만 다시 누르면 됩니다.

### ④ 이 PC에서 개발을 계속하려면 (선택)
- 현재 이 PC에 Node.js와 Python이 없어서 PowerShell 서버(`serve.ps1`)를 동봉했습니다.
- 더 편한 개발 환경을 원하면 https://nodejs.org 에서 Node.js LTS 설치를 권장합니다.
